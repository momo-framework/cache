# Обзор

`momo/cache` держит «тёплыми» дорогие результаты чтения на горячем пути 45k+ RPS
и — его отличительная черта — вытесняет их **реактивно** при изменении исходных
данных, слушая доменные события, а не полагаясь на угадывание TTL или
разбросанные ручные вызовы `forget()`.

## Слои

```
CacheInterface (Cache)         ← высокий уровень: remember(), теги, дженерики
        │  хеширует ключи через KeyHasherInterface (xxh3 / native)
        ▼
CacheStoreInterface (ArrayStore)  ← низкий уровень: TTL, индекс тегов
        ▲
QueryCache + #[Cached]          ← DX: кэш результата query-хендлера по атрибуту
        ▲
CacheInvalidationListener       ← реактивно: доменное событие → сброс тегов
   (подписан через InvalidationMap)
```

## Строительные блоки

| Концепция      | Тип                           | Ответственность                                 |
|----------------|-------------------------------|-------------------------------------------------|
| Фасад кэша     | `CacheInterface` / `Cache`    | `get/set/has/delete/clear/remember`, сброс тегов |
| Store          | `CacheStoreInterface` / `ArrayStore` | key/value с TTL и индексом тегов         |
| Запись         | `CacheEntry`                  | значение + абсолютный срок                       |
| Хешер ключей   | `KeyHasherInterface`          | xxh3-дайджест; нативное ускорение через фабрику   |
| `#[Cached]`    | атрибут + `CachedReader`      | декларативное кэширование на query-хендлере       |
| Кэш запросов   | `QueryCache`                  | применяет `#[Cached]`, выводит ключ из запроса    |
| Инвалидация    | `InvalidationMap` + `CacheInvalidationListener` | событие → теги → сброс        |

## Проектные решения

- **Реактивность вместо только-TTL.** TTL — запасной механизм; корректность даёт
  сброс тегов в момент, когда доменное событие сигнализирует об изменении.
  Результат с тегом `orders` живёт до события `OrderShipped` (или любого
  сопоставленного).
- **Дженерик `remember<T>()`.** Слот значения — единственный оправданный `mixed`
  в пакете (кэши гетерогенны); `remember()` дженерик, поэтому на местах вызова
  сохраняется типобезопасность. Кэшированный `null` — это попадание, не промах.
- **Инъекция часов.** Истечение TTL проверяется по внедрённому `ClockInterface`,
  поэтому детерминируемо в тестах (`MutableClock`).
- **Хеширование ключей за интерфейсом.** xxh3 через ext-hash по умолчанию;
  нативный крейт `momo_cache` может заменить его без изменения мест вызова
  (фабрика + проверка ABI + безопасный fallback, как у `StockCounterFactory`).
- **Кэширование запросов через атрибут.** `#[Cached(ttl, tags)]` + `QueryCache`
  дают эргономику в стиле Laravel, с тегами, напрямую заведёнными в реактивную
  инвалидацию.

## Корутино- и память-безопасность

`ArrayStore` держит состояние на время жизни воркера, не на запрос — нет
межзапросной утечки — и его методы не содержат точек приостановки I/O, поэтому
атомарны относительно соседних корутин в одном воркере Swoole. Свяжите
`CacheStoreInterface` на Redis/Swoole-Table для межворкерного шеринга; контракт
индекса тегов переносится.
